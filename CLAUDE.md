# Operating rules for Claude Code in alexsverdlik-site

Read this first at every session start.

## Repo identity
This is the public-facing marketing site at alexsverdlik.com. Static site generator: build-site.js produces dist/. Deploy mechanism: git push origin main → Netlify CD on the alexsverdlik-site Netlify project (separate from alex-sverdlik-dashboard-v2).

## Push approval
git add and git commit are auto-approved within this folder. git push origin main requires per-prompt approval from Naomi (chat-side review of the diff first). History-rewriting operations (rebase, force push, reset --hard on main) require explicit explanation and approval.

## Design preservation
The aesthetic is locked — luxury feel, Playfair Display + DM Sans, navy + gold + cream palette. Do NOT redesign. Extensions to the design system are fine for new components but should feel native to existing language. The blog post rendering CSS is the most distinctive layer; treat it as load-bearing.

## Build process
node build-site.js produces dist/. Always run a local build before commit to catch breakage. Never commit dist/ — it's git-ignored, regenerated by Netlify on deploy. The _redirects file lives at dist/_redirects and is generated by build-site.js (not hand-edited). 301 redirects must precede 200 rewrites in that file (first match wins in Netlify _redirects).

## Dependencies
Currently approved deps: whatever is in package.json today. New deps require explicit approval from Naomi (paste the use case + propose the smallest viable choice).

## Old dashboard is off-limits
The old production dashboard at /Users/naomischechter/Downloads/files (4)/dashboard-deploy/ is read-only reference for design extraction only. Do not edit, do not push to its repo, do not run its functions.

## Adjacent repos
The CRM backend lives in ~/Downloads/dashboard-v2 (45 Netlify Functions deployed at alex-sverdlik-dashboard-v2.netlify.app). Public site code (this repo) consumes those functions via fetch — chat widget POSTs leads, /testimonials build-time fetch from reviews-public-list. Cross-repo coordination happens by Naomi pasting context between sessions; no automatic sync.

## Per-edit review during onboarding
Naomi reviews diffs before approval until further notice. Batch operations (e.g., the multi-step build + verify + commit chain) are fine to propose under one approval, but file-write operations (Edit tool, Write tool) get individually previewed.

## Session handoffs
Build-log and next-Claude-prompt files live in Naomi's working folder, not this repo. End-of-session log updates happen in the chat session, not here.
